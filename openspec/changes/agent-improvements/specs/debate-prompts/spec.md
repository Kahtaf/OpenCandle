## ADDED Requirements

### Requirement: Bull researcher prompt builds the strongest case FOR the position

The bull researcher prompt SHALL instruct the LLM to build an advocacy case using evidence from all 5 prior analyst outputs visible in conversation context. It SHALL allow up to 2 tool calls for gap-filling.

#### Scenario: Bull prompt references analyst outputs

- **GIVEN** 5 analyst steps have completed for AAPL (their responses are in LLM context)
- **WHEN** the bull researcher prompt is generated
- **THEN** the prompt instructs the LLM to read the five analyst perspectives above
- **AND** the prompt requires citing analyst outputs and underlying tool evidence where available
- **AND** the prompt instructs the LLM to address any bearish signals and explain why they're less concerning

#### Scenario: Bull prompt allows limited tool calls (soft cap)

- **WHEN** the bull researcher prompt is generated
- **THEN** the prompt advises aiming for at most 2 tool calls to fill evidence gaps
- **AND** the prompt requires stating the gap BEFORE calling the tool
- **AND** the prompt requires reusing data already fetched in the session
- **AND** the limit is a prompt-level guideline, not a runtime-enforced constraint

#### Scenario: Bull prompt enforces structured ending

- **WHEN** the bull researcher prompt is generated
- **THEN** the prompt requires ending with "BULL THESIS: [2-3 sentences]"
- **AND** the prompt requires ending with "KEY RISK TO THIS THESIS: [one sentence]"

### Requirement: Bear researcher prompt dismantles the bull thesis

The bear researcher prompt SHALL instruct the LLM to attack the weakest assumptions in the bull case. It runs AFTER the bull researcher and SHALL reference both the 5 analyst outputs and the bull case visible in conversation context. It SHALL advise aiming for at most 2 tool calls for gap-filling (soft cap, not runtime-enforced).

#### Scenario: Bear prompt references bull case

- **GIVEN** 5 analyst steps and the bull researcher step have completed for AAPL
- **WHEN** the bear researcher prompt is generated
- **THEN** the prompt instructs the LLM to dismantle the bull thesis above
- **AND** the prompt requires attacking the weakest assumptions with cited evidence

#### Scenario: Bear prompt surfaces ignored negative data

- **WHEN** the bear researcher prompt is generated
- **THEN** the prompt instructs: "If the bull case ignored negative data points, surface them"

#### Scenario: Bear prompt enforces structured ending

- **WHEN** the bear researcher prompt is generated
- **THEN** the prompt requires ending with "BEAR THESIS: [2-3 sentences]"
- **AND** the prompt requires ending with "WHAT WOULD CHANGE MY MIND: [one sentence]"

### Requirement: Bull rebuttal prompt is self-gating and addresses bear arguments

The rebuttal prompt SHALL first instruct the LLM to check for BUY+SELL disagreement in the analyst signals visible in context. If no disagreement, respond with a one-line skip. Otherwise, address the bear's specific concerns. It SHALL NOT allow tool calls (reasoning-only).

#### Scenario: Rebuttal self-gates on consensus

- **WHEN** the rebuttal prompt is generated
- **THEN** the prompt begins with instructions to check the five analyst `SIGNAL:` lines specifically (not other mentions of BUY/SELL in bull/bear prose)
- **AND** instructs: if no BUY+SELL disagreement among those five signals, respond only with a line starting "REBUTTAL SKIPPED"

#### Scenario: Rebuttal requires concessions on split

- **WHEN** the rebuttal prompt is generated
- **THEN** the prompt instructs: "Concede any point where the bear is factually correct"
- **AND** instructs: "Do not repeat your original thesis — respond to the bear's NEW arguments"

#### Scenario: Rebuttal has no tool access

- **WHEN** the rebuttal prompt is generated
- **THEN** the prompt states: "No tool calls in the rebuttal. Work with existing evidence only."

#### Scenario: Rebuttal enforces structured ending (when not skipped)

- **WHEN** the rebuttal prompt is generated
- **THEN** the prompt requires ending with "CONCESSIONS: [bullet list]"
- **AND** the prompt requires ending with "REMAINING CONVICTION: [1-10]"

### Requirement: All debate prompts include execution guardrails

All debate prompts (bull, bear, rebuttal) SHALL include the `EXECUTION_GUARDRAILS` block — reuse prior tool outputs, stop quickly on unavailable data, don't retry failing calls.

#### Scenario: Guardrails present in all debate prompts

- **WHEN** the bull, bear, and rebuttal prompts are generated
- **THEN** each includes the EXECUTION_GUARDRAILS text

### Requirement: Debate prompts are standalone functions in orchestrator.ts

Debate prompts SHALL be generated by standalone `(symbol: string) => string` functions: `buildBullPrompt`, `buildBearPrompt`, `buildRebuttalPrompt`. They follow the same conventions as `ANALYST_PROMPTS` but are NOT keyed by `AnalystRole`.

#### Scenario: Functions follow existing pattern

- **GIVEN** `ANALYST_PROMPTS` is `Record<AnalystRole, (symbol: string) => string>`
- **WHEN** debate prompts are added
- **THEN** they are standalone exported functions, not a new Record type
- **AND** they use template literals, EXECUTION_GUARDRAILS, and structured endings
